## 技術的ルール（絶対厳守）

### Supabase / DB
- RLS は `SECURITY DEFINER` 関数でクロステーブル参照を解消する（`get_event_principals`, `is_event_artist`）。直接参照は無限再帰になる。
- `supabase db push` は `config.toml` の `schedule` キーをコメントアウトしないと CLI v2.84.2 でエラーになる（設定済み）。
- 関数の戻り値型を変える場合は `DROP FUNCTION` → `CREATE` の順。`CREATE OR REPLACE` では失敗する。
- `products.artist_id` は nullable（複数受取人のため）。

### Stripe
- Connect 承認権限は **admin（オーナー）のみ**。エージェントに委譲しない。
- 文言: 「プラットフォームオーナーによる口座開設審査」（「エージェントによる」は NG）。
- `verification_status`: `pending` → admin が `verified` or `rejected` に変える。

### QR / 決済フロー
- `/c/[qrCode]` ルートは `proxy.ts` で intl ミドルウェアをバイパスする（`NextResponse.next()` を先に返す）。
- `transactions.status` の許容値: `pending`, `processing`, `completed`, `succeeded`, `failed`, `refunded`。
- `transactions.sender_email` を必ず保存する（未登録ユーザーが後から登録した際の履歴紐付けのため）。

### Next.js
- `next/dynamic` with `ssr: false` は Server Component 内では使えない。`"use client"` が必要。
- `React.FormEvent` は非推奨。代わりに `React.SyntheticEvent<HTMLFormElement>` を使う。

### チェックイン
- カメラ QR 読み取りはネイティブアプリ側の責務。Web 版に実装しない。
